Contexte et justification

De nombreux individus, qu'ils soient étudiants, jeunes diplômés ou personnes sans emploi, rencontrent des difficultés pour trouver des opportunités de stage ou d'emploi adaptées à leur profil. Actuellement, la recherche se fait via plusieurs plateformes dispersées, ce qui rend le processus long et peu efficace. TalentLink vise à centraliser les offres, simplifier la mise en relation et offrir une chance équitable à tous en proposant un accompagnement personnalisé via une interface web moderne fonctionnant comme une application.

Objectifs de l'application

  • Permettre aux recruteurs partenaires de l'Institut Teccart de publier des offres (stage, emploi, alternance).
  • Permettre aux étudiants/candidats de créer un profil et de postuler directement.
  • Offrir un moteur de recherche filtré (domaine, localisation, compétences).
  • Proposer un système de recommandation basé sur le profil et les offres disponibles.
  • Intégrer une messagerie simplifiée entre recruteurs et candidats.
  • Ajouter un mode sombre / clair pour améliorer l'expérience utilisateur.
  • Fournir un tableau de bord analytique pour candidats et recruteurs.
  • Respecter les standards de l'Institut Teccart en matière de qualité et d'accessibilité.

🛠️ Description fonctionnelle

Interfaces clés
  • Espace candidat : création de CV en ligne, dépôt de candidatures, suivi (acceptée, en attente, refusée), portfolio (projets, GitHub, Behance, vidéos).
  • Espace recruteur : publication et gestion des offres, consultation de profils, statistiques (visibilité, candidatures reçues).
  • Recherche avancée : filtres par mots-clés, domaine, région, type de contrat.
  • Notifications : email et notifications internes pour nouvelles offres et réponses.
  • Tableau de bord analytique :
    • Pour les candidats : nombre de vues du profil, suivi des candidatures.
    • Pour les recruteurs : performances des offres publiées, nombre de candidatures reçues.
    • Pour l'Institut : statistiques globales, taux de placement, satisfaction.
  • Administration : modération, bannissement, suivi des statistiques globales.
  • Accessibilité & Responsive : utilisable sur ordinateur, tablette, mobile. Option mode sombre/clair.

⚙️ Contraintes techniques

  • Frontend : React.js (PWA, responsive) avec design system Institut Teccart.
  • Backend : FastAPI (Python) exposant une API REST sécurisée.
  • Base de données : SQLite (développement) → PostgreSQL (production).
  • Authentification : JWT avec gestion des rôles (étudiant, recruteur, admin Teccart).
  • Déploiement : Cloud (Heroku / Vercel) avec domaine Institut Teccart.
  • Messagerie interne : WebSocket pour communication temps réel.
  • Tests : tests unitaires & d'intégration, CI/CD avec GitHub Actions.
  • Sécurité : conformité RGPD, protection données étudiants, validation côté serveur.
  • Performance : temps de réponse < 2 secondes, optimisation mobile.

🗄️ Modèle de données

Entités principales :

  • Candidat : profil complet, compétences, formations, expériences
  • Recruteur : entreprise, secteur, offres publiées
  • Offre : détails poste, compétences requises, localisation
  • Candidature : liaison candidat-offre, statut, feedback
  • Message : communication interne, notifications
  • Administrateur : gestion plateforme, modération

Base de données relationnelle avec contraintes d'intégrité et index de performance.
